# 개인 콘텐츠 아카이브 경쟁 제품 분석 (2026년 7월 기준)

## 시장 배경: Pocket 종료가 만든 공백

- Mozilla는 **Pocket을 2025년 7월 8일 종료**했다. 이후 2025년 10월 8일까지 CSV 내보내기만 허용하고 전체 데이터를 영구 삭제했다. 약 2,000만 사용자, 누적 20억 건 이상의 저장 콘텐츠가 갈 곳을 잃었다.
- Kevin Rose(Digg)가 Pocket 인수를 공개 제안했으나 성사 확인은 없으며, Digg 자체도 2026년 1월 오픈 베타 후 3월에 베타를 접었다. **Pocket 브랜드 부활 가능성은 사실상 소멸.**
- 이 공백을 노리고 2025~2026년에 신생 툴이 대거 등장했고, 기존 툴(Raindrop, Readwise, Instapaper)이 반사이익을 얻었다. 2026년 리뷰들의 공통 진단: "read-it-later는 더 이상 수동 북마크가 아니라 **지식 수집 레이어(knowledge-ingestion layer)**가 됐다."

## 1. mymind (mymind.com)

- **핵심 기능**: 폴더/수동 태그 없는 "안티-조직화" 철학. 저장하면 AI가 알아서 이해·분류. 비주얼 카드 그리드 UI. 100% 프라이빗(공유·협업 기능 자체가 없음), 광고·트래킹 없음.
- **AI 활용**: AI 이미지 태깅, 이미지 텍스트 OCR, 지능형 북마크(페이지 유형 자동 인식 — 상품/레시피/트윗 등), 상위 플랜에서 AI 요약.
- **콘텐츠 보존**: **Article Backup(원문 전체 보존, 링크 사망 대비)은 상위 Mastermind 플랜($12.99/월) 전용.** 하위 플랜은 사실상 북마크+스냅샷 수준.
- **수집 경로**: 브라우저 확장(우클릭 저장 포함), iOS/Android/macOS 공유시트, 이메일 포워딩(remember@mymind.com — **플레인 텍스트만 지원**, 리치 텍스트/이미지 메일은 깨짐).
- **가격**: Student of Life $6.99/월, Mastermind $12.99/월(연간 결제 시 2개월 무료), Newton $299/년은 "출시 예정" 상태.
- **대표 불만**: (1) **내보내기(export)가 사실상 없음** — 2026년 초 기준 "제대로 된 export 옵션이 없다"는 평가, 이전 시 수동 재저장 필요. (2) 가격 대비 기능 논란. (3) 임포트/연동/오프라인/다국어 지원 부족. (4) 협업 불가. (5) 버그와 느린 업데이트. "폴더 없는 철학은 어느 순간까지만 통한다"는 회의론.

## 2. Raindrop.io

- **핵심 기능**: 가장 완성도 높은 전통형 북마크 매니저. 컬렉션(폴더)+태그, 그리드/리스트/헤드라인 뷰, 강력한 무료 티어. Pocket 난민의 1순위 무료 대안으로 꼽힘.
- **AI 활용**: **2026년 2월 "Stella" 출시** — 라이브러리 전체를 대상으로 자연어 질의응답, 자연어 명령으로 컬렉션 정리, 잊은 저장물 재발견. 자사 서버에서 처리. AI 태그 제안은 Pro 전용.
- **콘텐츠 보존**: Pro 전용 "웹 아카이브"(구 Permanent Copy) — 북마크 시점의 페이지 사본을 클라우드에 저장. 원본이 죽거나 유료화되어도 유지. 단, 공식 문서에 "아카이브 실패 필터·재시도" 안내가 있을 만큼 **실패 케이스가 구조적으로 존재**.
- **수집 경로**: 브라우저 확장, iOS/Android 앱 공유시트, 웹 앱, API. 소셜 포스트 전용 수집은 없음.
- **가격**: 무료(무제한 북마크) + Pro 연 $28~38(출처 간 상이 — 최근 인상 가능성, 확정치 미확인).
- **대표 불만**: 아카이브 실패, AI가 최근에야 추가됨(후발), 소셜 콘텐츠(로그인 벽 뒤 X/LinkedIn 포스트)는 아카이브 품질 보장이 안 됨. 레딧의 구체적 불만 스레드는 이번 검색에서 직접 확인 못 함(미확인 표시).

## 3. Karakeep (구 Hoarder)

- **핵심 기능**: 오픈소스 셀프호스팅 "bookmark everything" 앱. 링크·노트·이미지·PDF 저장, 전문(full-text) 검색, OCR 내장(스크린샷 검색 가능), **룰 엔진**(자동 태그/아카이브/정리 워크플로우), RSS 인제스트. GitHub 스타 24,800+, v0.31.0 (2026-02-22).
- **AI 활용**: OpenAI 또는 **로컬 Ollama 모델**로 자동 태깅. AI 모델 선택권을 사용자가 가짐.
- **콘텐츠 보존**: 페이지 콘텐츠 크롤링·보존(셀프호스팅이므로 저장 수준은 사용자 설정에 따름). "모든 방문 페이지를 아카이브"하려는 파워유저 논의가 GitHub에 존재.
- **수집 경로**: Chrome/Firefox 확장, iOS/Android 앱, RSS, **커뮤니티 Telegram 봇(karakeepbot)**, HN 업보트·레딧 세이브 동기화 툴(karakeep-sync) 등 생태계형 수집.
- **가격**: 셀프호스팅 완전 무료. **2026년 클라우드 베타: Free(북마크 10개/20MB 체험), Pro $4/월(북마크 5만 개/50GB, AI 태깅 포함)**, 기업 플랜 별도.
- **대표 불만**: 직접적인 불만 스레드는 이번 검색에서 확보 못 함(미확인). 구조상 명백한 진입장벽은 Docker 셀프호스팅 요구 — 클라우드 베타가 이를 해소하려는 시도. "2026년 가장 유력한 Pocket 후계자, 단 컨테이너를 돌릴 수 있다면"이라는 평가.

## 4. Fabric (fabric.so)

- **핵심 기능**: "생각하는 AI 워크스페이스" 포지셔닝. 링크뿐 아니라 **파일(이미지·비디오·PDF·오디오·문서)을 폭넓게 저장**하는 드라이브+북마크 하이브리드. Google Drive/Dropbox 연동. 사용자 30만+ 주장.
- **AI 활용**: 저장물 자동 분석·자동 태깅, 관련 항목 간 연관(knowledge graph식 연결), 자연어 시맨틱 검색, AI 어시스턴트 채팅.
- **콘텐츠 보존**: 확장으로 이미지·텍스트·**전체 페이지 캡처(자동 스크롤 풀페이지 스크린샷)** 지원. 파일은 원본 저장(전송·저장 시 암호화). 다만 웹페이지의 "HTML 원문 아카이브" 수준인지는 미확인 — 문서상으로는 스크린샷 중심.
- **수집 경로**: 브라우저 확장, iOS/Android 공유시트, 데스크톱 앱, 드래그&드롭, Android 스크린샷 자동 수집.
- **가격**: 무료 플랜 + 프리미엄 $8/월부터.
- **대표 불만**: 레딧/HN에서 유의미한 불만 스레드를 이번 검색에서 확보 못 함(미확인). 참고로 경쟁사 비교글(Saner.AI 등)에서 대안 수요가 존재한다는 정황은 있음.

## 5. Recall (getrecall.ai)

- **핵심 기능**: "자기조직화(self-organizing) AI 지식베이스". 아티클·**YouTube·팟캐스트**·PDF·노트를 저장하면 요약+자동 분류+지식그래프 연결. **간격 반복(spaced repetition) 퀴즈**로 기억 정착까지 담당 — 학습 지향이 차별점. 사용자 50만+, 2026년 "Recall 2.0" 출시.
- **AI 활용**: 자동 요약, 자동 카테고리화, 관련 콘텐츠 자동 연결(지식그래프), 저장물 대상 챗.
- **콘텐츠 보존**: 요약+본문 저장 중심(원문 페이지의 완전 아카이브 여부는 미확인). PDF는 100MB까지.
- **수집 경로**: Chrome/Firefox 확장, iOS/Android, 웹.
- **가격**: 무료 Lite / Plus $10/월(연간 결제) / Max $38/월.
- **대표 불만**: PDF **1개씩만 업로드 가능**(경쟁사는 10개+ 배치), 비영어권 언어 지원 부족(인터페이스·출력이 영어 기본), 자동 생성 그래프가 산만함, 데스크톱 단축키·임포트 옵션 부족.

## 6. Readwise Reader

- **핵심 기능**: 파워리더용 read-it-later 최강자. 웹 아티클·PDF·EPUB·뉴스레터·RSS·**YouTube(시간동기 트랜스크립트 하이라이트)**·X 스레드를 하나의 인박스로. 하이라이트→Obsidian/Notion/Roam 내보내기 파이프라인, 일일 리뷰(간격 반복). Pocket 종료 후 "최고의 read-it-later"로 반복 언급.
- **AI 활용**: "Ghostreader" — 문서 인지형 AI 어시스턴트(요약, 질의, 프롬프트).
- **콘텐츠 보존**: 저장 시점에 파싱된 본문 텍스트 보존. **유료 페이월 콘텐츠는 "보이는 것만" 저장.** 완전한 페이지 스냅샷(웹 아카이브)이 아니라 리딩용 파싱 텍스트 중심.
- **수집 경로**: 확장, 모바일 공유시트, **전용 이메일 주소(뉴스레터 구독/포워딩)**, RSS, 파일 업로드.
- **가격**: **무료 티어 없음**(30일 체험). $9.99/월(연간) / $12.99/월(월간). Lite 티어 $5.59/월. 학생 50% 할인.
- **대표 불만**: 무료 티어 부재와 연 $120의 가격, 기능·설정 과다로 온보딩 부담, 캐주얼 유저에게 과잉.

## 7. Anytype

- **핵심 기능**: 로컬-퍼스트 + E2E 암호화 오픈소스 PKM(노션 대체재 성격). 객체/타입/관계 기반 모델링, 그래프 뷰. 아카이브 전용 툴이 아니라 범용 지식베이스.
- **AI 활용**: 2026년 로드맵에서 "무AI/로컬AI/클라우드AI/하이브리드를 사용자가 선택"하는 방향 표명 — **AI 자동 분류는 아직 핵심 기능이 아님.**
- **콘텐츠 보존**: 웹 클리퍼로 전체 페이지/하이라이트 저장 가능하나, **클리퍼가 페어링 실패·내용 누락(제목/링크만 저장됨) 등 신뢰성 문제로 악평**(Chrome 스토어 리뷰).
- **수집 경로**: 웹 클리퍼(Chrome), 데스크톱/모바일 앱. 공유시트·이메일·봇 같은 다변화된 수집은 약함.
- **가격**: 코어 앱 전 기능 무료(로컬 100MB 원격 저장) + Plus $4 / Pro $8 / Ultra $16/월(원격 저장 용량 확대).
- **대표 불만**: 멀티디바이스 동기화 지연(30~90초)·충돌·신뢰성 문제(가장 흔한 불만), iOS 랙·Android 크래시(대용량 라이브러리), 협업 기능 빈약, Notion 대비 완성도 부족.

## 이 시장에서 비어있는 포지션 (기획 시사점)

**1. "소셜 포스트의 본문 보존 + 범용 아카이브"의 결합은 아직 아무도 못 했다.**
- dewey는 소셜(X/LinkedIn) 동기화는 잘하지만 범용 웹/파일 아카이브가 아니고, 과금·동기화 불만이 크다.
- Raindrop/mymind/Readwise는 범용이지만 로그인 벽 뒤의 LinkedIn/X 포스트를 온전히 보존하지 못하거나 보존을 최상위 플랜에 가둔다.
- Kiroku처럼 해시까지 남기는 증거급 보존은 트윗 전용 단일 기능 툴에 머문다.
- → **"LinkedIn/X 포스트를 텍스트+이미지+작성자 메타데이터까지 원문 보존하고, 일반 링크·파일과 한 아카이브에서 AI로 조직화"**가 정확히 빈 자리.

**2. 콘텐츠 보존이 기본값인 툴이 없다.**
- 보존은 유료 상위 티어의 프리미엄 기능(mymind Mastermind, Raindrop Pro, Instapaper Premium)이거나 셀프호스팅 요구(Karakeep). Pocket 데이터 영구 삭제 사건 이후 "내 저장물은 서비스가 죽어도 내 것"이라는 불안이 시장 전체에 깔려 있다. **보존+완전한 export 보장을 기본값·신뢰 서사로 삼는 포지션**이 유효하다. (mymind의 export 부재, dewey의 유료 Export Pass가 반면교사.)

**3. 음악·미디어 파일은 전 제품의 사각지대.**
- YouTube는 Readwise(트랜스크립트)·Recall(요약)이 다루지만, **음악 링크(Spotify 등)·오디오·일반 미디어 파일을 1급 객체로 다루는 아카이브 툴은 확인되지 않음**(Fabric이 파일 저장은 하나 음악 특화 기능은 없음). 멀티미디어 전반을 다루겠다는 아카식 레코드 컨셉과 정합.

**4. AI 자동 태깅은 이미 표준(table stakes), 차별화는 그 위층.**
- 자동 태깅·시맨틱 검색은 Raindrop, Karakeep, Fabric, mymind, Recall, 신생 북마크 앱 전부가 제공. 차별화 여지는 (a) 아카이브 전체와의 대화(Stella/Recall이 시작), (b) 자동 '체계화'(컬렉션·구조 생성 — Recall의 그래프는 "산만하다"는 불만), (c) 크로스 콘텐츠 연결의 품질.

**5. 비영어권(한국어 포함) 지원은 명확한 공백.**
- Recall(언어 지원 불만), mymind(다국어 부족)에서 반복 확인. 한국어 콘텐츠의 AI 분류·요약 품질을 보장하는 제품은 사실상 없음.

**6. 가격 벤치마크.**
- 심리적 저항선: 무료 티어 없는 Readwise($120/년)는 지속 불만, Karakeep Cloud $4/월·Fabric $8/월·Recall $10/월이 중간대. **무료 티어 + $5~10/월 유료**가 시장 기대치. 보존(스토리지 비용 발생) 기능을 유료화 축으로 쓰되 mymind처럼 최상위에 가두면 핵심 가치 전달이 안 되는 딜레마 주의.

### 확인하지 못한 사항 (명시)
- Raindrop·Karakeep·Fabric의 레딧/HN 구체 불만 스레드 원문(검색 결과 미포착 — 공식 문서·리뷰 사이트 기반 간접 확인만 수행).
- Fabric의 HTML 원문 아카이브 여부(풀페이지 스크린샷은 확인, DOM 보존은 미확인).
- Raindrop Pro 정확한 현재 연간 가격($28 vs $38 출처 충돌), Instapaper Premium 정확한 현재 가격($3.99 vs $5.99 출처 충돌).
- Digg의 Pocket 브랜드 인수 성사 여부(제안만 확인, 성사 확인 없음).
- Recall Max 티어($38/월)의 정확한 기능 구성.

## Key Takeaways
- Pocket 종료(2025.7.8, 데이터 영구 삭제)로 2천만 사용자 공백이 생겼고, 이후 시장 전체에 '서비스가 죽어도 내 데이터는 안전한가'라는 신뢰 문제가 핵심 구매 요인으로 부상 — 보존+완전 export 보장을 기본값으로 내세우는 것이 강력한 차별화 서사
- 콘텐츠 원문 보존은 현재 전부 프리미엄 상위 티어(mymind Mastermind $12.99, Raindrop Pro, Instapaper Premium)이거나 셀프호스팅(Karakeep) 요구 — '보존이 기본값'인 클라우드 제품은 빈 자리
- LinkedIn/X 등 로그인 벽 뒤 소셜 포스트의 본문 보존 + 범용 웹/파일 아카이브를 한 제품에 결합한 곳이 없음 — dewey(소셜 전용, 불만 다수)와 Raindrop(범용, 소셜 취약) 사이의 공백이 아카식 레코드의 최적 포지션
- AI 자동 태깅·시맨틱 검색은 2026년 기준 이미 표준 기능(table stakes) — 차별화는 아카이브 전체와의 대화(Raindrop Stella가 2026.2 시작), 자동 구조/컬렉션 생성, 콘텐츠 간 연결 품질에서 이뤄져야 함
- 음악(Spotify 등)·오디오·미디어 파일을 1급 객체로 다루는 아카이브 툴은 확인되지 않음 — 멀티미디어 전방위 수집이라는 컨셉 자체가 차별점
- 수집 경로는 확장+모바일 공유시트가 기본이고, 차별화 요소는 이메일 포워딩(Readwise/mymind)과 메신저 봇(SaveDay의 Telegram, MarkIt의 WhatsApp) — Karakeep 생태계처럼 봇/자동 동기화 수집이 파워유저 견인 요소
- 가격 기대치는 무료 티어 + 월 $5~10 (Karakeep Cloud $4, Fabric $8, Recall $10) — 무료 티어 없는 Readwise($120/년)는 지속적 불만 대상이며, 보존 기능을 최상위 티어에 가둔 mymind는 핵심 가치 전달 실패 사례
- 경쟁사 대표 약점: mymind(export 부재·잠금), Raindrop(아카이브 실패 케이스), Recall(PDF 1개씩 업로드·비영어 지원 부족), Readwise(페이월은 '보이는 것만' 저장·가격), Anytype(클리퍼·동기화 신뢰성), dewey(export 유료화·동기화 불안정) — 각각이 기획 체크리스트
- 한국어 등 비영어권 AI 분류·요약 품질을 보장하는 제품이 사실상 없음 — 한국 시장 우선이라면 명확한 로컬 우위 확보 가능
- 가장 위협적인 경쟁자는 Karakeep(오픈소스 24.8k stars + 2026년 클라우드 $4/월 진출)과 Raindrop(무료 티어 + Stella AI) — 전자는 가격·데이터 소유권, 후자는 완성도·설치 기반에서 우위
